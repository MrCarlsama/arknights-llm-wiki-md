# LLM Wiki Markdown Directory

这个目录给大模型读，不给程序搜索。目标是让模型凭语义理解选择路径，而不是依赖硬编码查询。

## 推荐阅读路径

1. `index.md`: 总入口和运行时边界。
2. 本文件: 目录职责、文件规模、证据层级。
3. `aliases.md`: 仅在用户问题含俗称、称号、疑似错称时用于找候选页面。
4. 目标主题对应目录的 `INDEX.md`。
5. INDEX 指向的具体 Markdown 页面。
6. 页面内部链接指向的证据页。

## 证据层级

| 层级 | 文件夹 | 用途 |
| --- | --- | --- |
| L1 已整理剧情 | `lore/storylines/` | 剧情线主 wiki，适合作为剧情问答第一入口 |
| L2 剧情分段笔记 | `lore/story-notes/` | 长剧情按活动/章节拆开的分段笔记 |
| L3 剧情导航 | `storylines/` | 剧情源顺序、章节导航、证据跳转 |
| L4 角色实体 | `characters/` | 角色档案、别名、剧情引用 |
| L4 单位实体 | `units/` | 干员/单位数据与游戏实体信息 |
| L4 NPC | `npcs/` | 非干员人物实体 |
| L4 活动实体 | `events/` | 活动、章节、剧情线实体 |
| L4 阵营组织 | `factions/` | 国家、组织、势力 |
| L4 概念 | `concepts/` | 世界观概念与主题词 |
| L5 关系 | `relations/` | 实体之间的关系页面 |
| L6 剧情原文证据 | `sources/story/` | 内部化剧情原文/简介证据页 |
| L6 游戏数据证据 | `sources/json/` | 内部化游戏数据证据页 |

## 回答约束

- 如果没有打开足够 Markdown 页面，不要回答确定结论。
- 如果只能凭外部常识猜测，必须标明“当前 wiki 证据不足”。
- 昵称、俗称、黑话如果没有内部别名或页面证据，不得推断成确定人物。
- 如果引用证据，只引用内部 `wikiPath`。
- 不展示外部 URL，不展示 `source_uri`，不展示 `source_id`。
